# FLOR MUSIC

Бесплатный музыкальный плеер для поиска и прослушивания музыки. Сделан из готового дизайна и полностью рабочий — **без API-ключей, без регистрации и без оплаты**.

## Как запустить

Нужен установленный [Node.js](https://nodejs.org) (уже есть на вашем ПК).

```bash
npm start
```

Затем откройте в браузере: **http://localhost:5173**

> Важно: открывать именно через сервер (`npm start`), а не двойным кликом по `index.html` — браузеры блокируют сетевые запросы и ES-модули с `file://`.

## Что умеет

- 🔎 **Поиск** треков и исполнителей по огромному каталогу.
- ▶️ **Реальное воспроизведение** через `<audio>`: play/pause, перемотка, громкость, следующий/предыдущий, перемешать, повтор (вкл/все/один).
- ❤️ **Любимое** и **история** прослушивания — сохраняются в браузере (localStorage).
- 🏠 **Главная** с живыми трендами и плейлистами.
- 🖼️ Реальные обложки альбомов.
- 🌗 Светлая/тёмная тема, полноэкранный плеер, очередь, медиа-клавиши.

## Бесплатные источники музыки

| Источник | Что даёт | Ключ нужен? |
|----------|----------|-------------|
| **Audius** (`api.audius.co`) | Полные треки целиком, трендовые чарты, плейлисты | Нет |
| **iTunes Search API** | Мейнстрим-каталог, обложки, превью 30 сек | Нет |

Оба API публичные, бесплатные и работают напрямую из браузера (CORS).

## Структура

```
app/
  index.html       # разметка (на основе дизайна FLOR)
  styles.css       # дизайн-система + дополнения
  js/
    icons.js       # SVG-иконки и логотип
    api.js         # слой музыкальных API (Audius + iTunes)
    player.js      # аудио-движок (очередь, перемотка, громкость)
    app.js         # логика приложения и рендер экранов
  assets/          # логотип
server.mjs         # локальный статический сервер (без зависимостей)
```

## Ограничения

- Треки из **iTunes** — это официальные превью по 30 секунд (ограничение Apple).
- Треки из **Audius** играют целиком.
- Каталог Audius — это в основном независимые артисты; мейнстрим-хиты ищите во вкладке **iTunes**.
